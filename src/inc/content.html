Front End Performance

What is this about?

Perfomance is the practice of optimizing your web prject to deliver the quickest rendering possible to your users. This can include everything from assets like styles and/or scripts, to limiting requests and lazy loading images. Google explains it here in their article called Optimizing Performance:

Performance is a feature. Deliver your content to users as quickly as possible. Once they're in your app, make page interaction and rendering as smooth as possible.

Why is this important?

Perfomance is a hugely important factor for any type of web project. Publications with users who access their content using various devices with different network connections may find this feature becomes increasingly critical to the success of their web products.

What have we done so far?

Modern Healthcare has implemented some tools and methodologies based on what we've seen with modern websites. These include:

  • Streaming build system
  • CSS preprocessor
  • DRY-ing out CSS, don't-repeat-yourself methodology
  • BEM, block-element-modifier metholody for CSS
  • Concatonating and compressing scripts
  • Compressing image assets

What can we improve on?

While we have made some major steps, there are many other aspects we'd like to implement into our development and production:

  • CDN
  • Cache control
  • Minify-ing markup
  • Lazy loading non-essential content (based on largeer screen sizes)
  • Removing block rendering scripts
  • Limiting requests
  • Inlining critical styles and scripts (perceived performance)
  • Testing selector specifity
  • HTTP/2
